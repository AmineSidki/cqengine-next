# Contributing

Contributions are welcome! This fork aims to maintain backward compatibility while adding modern features.

## Guidelines

Some general guidelines to follow when contributing:

1. **Preserve backward compatibility** - All existing APIs must continue to work
2. **Add @author attribution** - Add your name to modified files
3. **Follow Keep a Changelog** - Update `CHANGELOG.md` with your changes
4. **Run tests before committing**: `mvn clean test`
5. **Include Docker tests** if modifying persistence/index code
6. **Update documentation** as needed
7. **Submit a pull request** with a clear description of your changes
8. **Be respectful and collaborative** in code reviews and discussions
9. **Adhere to coding standards** - Follow existing code style and conventions
10. **Write clear commit messages** - Describe what and why, not how
11. **Test thoroughly** - Ensure your changes are well-tested
12. **Engage with the community** - Participate in discussions and help others
13. **Report issues** - Use the issue tracker for bugs and feature requests
14. **Stay updated** - Keep your fork in sync with the main repository
15. **Seek help when needed** - Don't hesitate to ask for guidance or clarification
16. **Respect licensing terms** - Ensure compliance with all relevant licenses
17. **Document code changes** - Update inline comments and documentation as necessary
18. **Use feature branches** - Create separate branches for each feature or bug fix
19. **Review pull requests** - Provide constructive feedback on others' contributions
20. **Celebrate contributions** - Acknowledge and appreciate the efforts of all contributors
